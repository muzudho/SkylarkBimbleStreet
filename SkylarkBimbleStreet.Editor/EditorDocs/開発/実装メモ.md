# 実装メモ

作成日: 2026-06-14

## 現在の実装範囲

`SkylarkBimbleStreet.Editor` は MonoGame ベースの別プロジェクト。
ゲーム本体と同じ `1920 x 1080` の仮想座標を使い、ゲーム本体の `Stages/stage-*.json` を直接読み書きする。

実装済み。

- `stage-001.json` から `stage-007.json` の読み込み。
- ステージ全体の表示。
- 壁、宝石、符牒の破片、出口、バス停、病院、障害物、障害物の移動範囲、プレイヤー開始位置の表示。
- 左クリック選択。
- ドラッグ移動。
- `PageUp` / `PageDown` キーでステージ切り替え。
- `S` キー保存。保存前に `Stages/Backups` へバックアップ作成。
- `R` キー再読み込み。
- `G` キーで 40px グリッドスナップ切り替え。
- `Delete` キーで選択中の壁、収集物、障害物を削除。
- 障害物ドラッグ時に、移動軸に対応する `min` / `max` も同じ量だけ移動。

## 主なファイル

- `Program.cs`: エディター起動入口。
- `EditorGame.cs`: 描画、入力、選択、ドラッグ、保存、読み込み。
- `StageData.cs`: JSON 読み書き用 DTO。
- `SkylarkBimbleStreet.Editor.csproj`: MonoGame DesktopGL 参照。

## 制限

- 矩形の追加とサイズ変更はまだない。
- 削除できるのは壁、収集物、障害物のみ。必須要素のプレイヤー開始位置、出口、バス停、病院は削除しない。
- 障害物の `min` / `max` は障害物ドラッグに追従する。移動範囲のサイズ変更や端点の直接編集はまだできない。
- アイテムは `Items: [{ Kind, Bounds }]` 形式で扱う。`Kind` は現時点で `ticketPiece` と `gem`。
- 旧 `Collectibles` 形式は読み込み時にゲーム側と同じ規則で `Items` へ変換する。保存すると `Items` 形式になる。
- 文字描画は使っていない。状態表示はウィンドウタイトルに寄せている。

## 次に足す候補

1. オブジェクト種類を選んで新規追加できるようにする。
2. 右下ハンドルでサイズ変更できるようにする。
3. 障害物の移動範囲端点、またはサイズを編集できるようにする。
4. アイテム種類の切り替えや新規アイテム追加をできるようにする。
