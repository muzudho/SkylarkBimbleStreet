# マップエディター操作説明書

SkylarkBimbleStreet.Editor は、ゲーム本体の `Stages/stage-001.json` から `stage-007.json` を読み込み、ステージ上の矩形オブジェクトを見ながら移動できる最小マップエディター。

## 起動する

リポジトリのルートで次を実行する。

```powershell
dotnet run --project .\SkylarkBimbleStreet.Editor\SkylarkBimbleStreet.Editor.csproj -c Debug
```

Visual Studio から起動する場合は、`SkylarkBimbleStreet.Editor` プロジェクトをスタートアップにして実行する。

## 編集対象

エディターは次の JSON ファイルを直接読み書きする。

```text
SkylarkBimbleStreet/Stages/stage-001.json
SkylarkBimbleStreet/Stages/stage-002.json
...
SkylarkBimbleStreet/Stages/stage-007.json
```

保存するとゲーム本体のステージデータが変わる。保存後にゲームを起動し直すと、変更後の配置で遊べる。

## 画面の見方

- 灰色: 壁。
- 紫: 符牒の破片。JSON の `Items[].Kind` が `ticketPiece` のもの。
- 黄色: 宝石。JSON の `Items[].Kind` が `gem` のもの。
- 緑: 出口。
- 青: バス停。
- 白と青枠: 病院。
- 赤: 障害物。
- 赤い細枠: 障害物の移動範囲。
- 水色: プレイヤー開始位置。
- 白い太枠: 現在選択中のオブジェクト。
- オレンジの塗りと枠: オブジェクト同士が重なっている場所。

ウィンドウタイトルには、現在の JSON ファイル名、ステージ名、選択中の種類、保存や読み込みの状態が表示される。

## 基本操作

| 操作 | 内容 |
| --- | --- |
| 左クリック | オブジェクトを選択する。 |
| 左ドラッグ | 選択したオブジェクトを移動する。 |
| `PageUp` | 前のステージへ切り替える。 |
| `PageDown` | 次のステージへ切り替える。 |
| `S` | 現在のステージ JSON を保存する。保存前に `Stages/Backups` へバックアップを作る。 |
| `R` | 現在のステージ JSON を再読み込みする。保存していない移動は破棄される。 |
| `G` | 40px グリッドスナップのオン、オフを切り替える。 |
| `T` | 選択中のアイテムを宝石と符牒の破片で切り替える。 |
| `1` | マウス位置に壁を追加する。 |
| `2` | マウス位置に宝石を追加する。 |
| `3` | マウス位置に符牒の破片を追加する。 |
| `4` | マウス位置に障害物を追加する。 |
| `Delete` | 選択中の壁、アイテム、障害物を削除する。 |
| `Esc` | エディターを終了する。 |

## 編集の流れ

1. エディターを起動する。
2. `PageUp` / `PageDown` で編集したいステージへ移動する。
3. 動かしたいオブジェクトを左クリックで選ぶ。
4. 左ドラッグで位置を調整する。
5. 必要なら `G` キーで 40px グリッドスナップを切り替える。
6. 必要なら `1` から `4` で壁、アイテム、障害物を追加する。
7. アイテムの種類を変えたい場合は選択して `T` キーを押す。
8. 不要な壁、アイテム、障害物は選択して `Delete` キーで削除する。
9. `S` キーで保存する。
10. ゲーム本体を起動し、配置が意図通りか確認する。
## 注意点

- 保存すると JSON ファイルが直接上書きされるが、保存前の内容は `Stages/Backups` にコピーされる。
- 矩形のサイズ変更はまだできない。
- `Delete` キーで削除できるのは壁、アイテム、障害物。プレイヤー開始位置、出口、バス停、病院は削除できない。
- 障害物をドラッグすると、移動方向に対応する `min` / `max` も同じ量だけ移動する。移動範囲のサイズ変更はまだできない。
- 重なりがあるとウィンドウタイトルに `overlaps` 件数が出る。意図しない重なりはオレンジ表示を目印に直す。
- アイテムは JSON 上では `Items` にまとまっており、`Kind` で符牒の破片や宝石を見分ける。旧 `Collectibles` 形式も読み込み時に `Items` へ変換される。

## 困ったとき

保存前なら `R` キーで現在の JSON を再読み込みすると、未保存の移動を破棄できる。

保存後に戻したい場合は、Git の差分で `SkylarkBimbleStreet/Stages/*.json` の変更を確認して戻すか、`SkylarkBimbleStreet/Stages/Backups` のバックアップ JSON を参照する。
