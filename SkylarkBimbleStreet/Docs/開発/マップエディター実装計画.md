# マップエディター実装計画

作成日: 2026-06-13

## 目的

`StageDefinitions.cs` に数値を直接書いてステージを調整する負担を減らす。
ゲーム本体はステージデータを JSON から読み込み、別プロジェクトのマップエディターで配置を見ながら編集できるようにする。

## 現状

- ステージ定義は `Stages/stage-001.json` から `Stages/stage-009.json` までの JSON ファイルへ移行済み。
- `Stage` は、名前、開始位置、出口、バス停、病院、背景色、壁、アイテム、障害物を持つ。
- アイテムは `Items: [{ Kind, Bounds }]` 形式へ移行済み。旧 `Collectibles` は読み込み互換として残している。
- `StageDefinitions.cs` は `StageLoader.LoadStages()` を呼ぶだけの入口になっている。

## 方針

- まずゲーム本体が JSON を読めるようにする。
- 既存ステージを JSON に移して、現行ステージがそのまま遊べることを確認する。
- エディターはそのあとに別プロジェクトとして追加する。
- 最初のエディターは高機能にしない。表示、選択、ドラッグ移動、保存、読み込みを優先する。
- ゲーム本体とエディターの座標系は同じ `1920 x 1080` にする。

## データ形式案

ステージごとに 1 JSON ファイルを置く。

```text
SkylarkBimbleStreet/
  Stages/
    stage-001.json
    stage-002.json
    stage-003.json
```

JSON の大枠は以下。

```json
{
  "name": "Stage 1",
  "playerStart": { "x": 95, "y": 95 },
  "exitBounds": { "x": 1770, "y": 930, "width": 92, "height": 92 },
  "busStopBounds": { "x": 64, "y": 210, "width": 176, "height": 124 },
  "hospitalBounds": { "x": 64, "y": 64, "width": 176, "height": 126 },
  "backgroundColor": { "r": 18, "g": 22, "b": 31 },
  "walls": [
    { "x": 0, "y": 0, "width": 1920, "height": 38 }
  ],
  "items": [
    { "kind": "ticketPiece", "bounds": { "x": 390, "y": 250, "width": 34, "height": 34 } },
    { "kind": "gem", "bounds": { "x": 660, "y": 820, "width": 34, "height": 34 } }
  ],
  "hazards": [
    {
      "bounds": { "x": 340, "y": 500, "width": 64, "height": 64 },
      "velocity": { "x": 0, "y": 250 },
      "min": 410,
      "max": 760
    }
  ]
}
```

## 実装ステップ

1. [完了] `StageData` 系の読み込み用 DTO を追加する。
   - `StageData`, `RectangleData`, `Vector2Data`, `ColorData`, `HazardData` を用意する。
   - MonoGame 型に依存しすぎない素直な JSON 用クラスにする。

2. [完了] `StageLoader` を追加する。
   - `Stages/*.json` を名前順で読み込む。
   - JSON から `Stage` と `Hazard` を生成する。
   - 読み込み失敗時は、どのファイルのどの項目が問題か分かる例外にする。

3. [完了] 既存ステージを JSON へ移す。
   - `StageDefinitions.cs` の値を `Stages/stage-001.json` 以降へ移す。
   - 当面は `StageDefinitions.CreateStages()` を残し、内部で `StageLoader` を呼ぶ形にすると変更範囲が小さい。
   - JSON ファイルはビルド出力へコピーされるように `.csproj` に登録する。

4. [一部完了] ゲーム本体の読み込み確認をする。
   - Debug と Release のビルド確認。
   - ステージセレクトのカード、ミニマップ、宝石数、障害物数が以前と同じように出ることを確認する。
   - 少なくとも Stage 1 を実プレイして、符牒の破片、宝石、出口開放、クリアを確認する。

5. [完了] エディター用プロジェクトを追加する。
   - `SkylarkBimbleStreet.Editor` を別プロジェクトとして作る。
   - 最初は MonoGame ベースを候補にする。ゲームと同じ座標系、同じ見た目で確認しやすいため。
   - ソリューションにゲーム本体とエディターを並べる。

6. [一部完了] エディターの最小機能を作る。
   - JSON 読み込み。
   - ステージ全体の表示。
   - 壁、アイテム、出口、バス停、病院、障害物の表示。
   - マウスで選択。
   - ドラッグ移動。
   - JSON 保存。

7. [一部完了] エディターの次段階を作る。
   - [完了] 新規矩形の追加。
   - [完了] 選択物の削除。
   - 矩形サイズ変更。
   - [完了] グリッドスナップ。
   - [完了] 障害物の移動範囲プレビュー。
   - [完了] 障害物の移動範囲編集。
   - [完了] 符牒の破片と宝石の JSON 上の明示。
   - [完了] アイテム種類の切り替え。


## 最小エディターの現状

`SkylarkBimbleStreet.Editor` プロジェクトを追加済み。
MonoGame ベースで、ゲーム本体と同じ `1920 x 1080` 座標系のステージ表示を行う。

現在できること。

- `Stages/stage-001.json` から `Stages/stage-009.json` を読み込む。
- 壁、宝石、符牒の破片、出口、バス停、病院、障害物、障害物の移動範囲、プレイヤー開始位置を表示する。
- 左クリックで矩形を選択する。
- ドラッグで選択した矩形を移動する。
- `PageUp` / `PageDown` キーでステージを切り替える。
- `S` キーで JSON 保存する。保存前に `Stages/Backups` へバックアップを作る。
- `R` キーで現在ステージを JSON から再読み込みする。
- `G` キーで 40px グリッドスナップを切り替える。
- `Delete` キーで選択中の壁、アイテム、障害物を削除する。
- 障害物をドラッグすると、移動軸に対応する `min` / `max` も同じ量だけ移動する。
- 選択中障害物の移動範囲端点を `A` / `D` / `J` / `L` キーで編集する。
- `T` キーで選択中アイテムの `Kind` を `gem` / `ticketPiece` で切り替える。
- 1 壁、2 宝石、3 符牒の破片、4 障害物をマウス位置に追加する。
- 画面左上の追加ツールバーで壁、宝石、符牒の破片、縦移動障害物、横移動障害物を選び、空き場所クリックで追加する。
- オブジェクト同士が重なっている交差部分と関係オブジェクトをオレンジで警告表示する。

現時点では、サイズ変更は右下ハンドルで実装済み。追加するオブジェクト種類の画面内選択も左上ツールバーで実装済み。

## 推奨判断

- 現行ステージは `Items: [{ Kind, Bounds }]` 形式へ移行済み。
- 旧 `Collectibles` は移行期間の読み込み互換として残す。
- エディターはまず MonoGame で作る。ゲーム内表示とズレにくく、追加依存も少ない。
- JSON には `version` を入れておくと後で移行しやすい。
