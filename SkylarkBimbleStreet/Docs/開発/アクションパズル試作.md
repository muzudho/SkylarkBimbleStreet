# アクションパズル試作

記録日: 2026-06-10

## 概要

`Game1.cs` に、外部アセットなしで動く 1画面アクションパズル試作を実装した。

## ゲーム内容

- 青いプレイヤーを動かす。
- 黄色い宝石をすべて集める。
- 宝石をすべて集めると、右下の緑の出口がクリア条件になる。
- 赤い障害物に当たるとスタート位置へ戻る。
- `R` または Xbox コントローラー Start でリトライ。
- `Esc` または Xbox コントローラー Back で終了。

## 1ステージ内の遊びの核

- [必須] この試作の中心は、障害物の危険圏に踏み込み、宝石を取ったらすぐ離脱する、短いリスクとリターンの管理に置く。
- [必須] 宝石は安全すぎる場所だけに置かず、取るために障害物との距離、タイミング、逃げ道を判断する配置にする。
- [必須] 障害物は理不尽に避けられないものではなく、待つ、近づく、取る、離れるという判断と操作で対処できるものにする。
- [必須] 良いステージは、「今取りに行くか」「一度待つか」「取ったあとどちらへ逃げるか」「もう1個続けて取るか」をプレイヤーに考えさせる。
- [必須] 制限時間を付けず、コントローラーを離してもミスにならないようにする。プレイヤーが安全な場所で止まり、障害物の動きと宝石への入り方を落ち着いて考えられることを良い点として残す。
- [必須] 障害物は自分から襲い掛かってくる敵ではなく、プレイヤーが近づいたときに危険になる盤面要素として扱う。リアルタイムに動く盤面を観察し、プレイヤー自身がリスクを取る瞬間を選ぶゲームにする。
- [必須] プレイヤーが立ち止まって考え、タイミング、一気に行くコース、どこで休憩を取るかを判断できるステージにする。休憩地点を自分の頭で見つけられることを面白さにする。
- [必須] 目押しが得意、レバー操作が得意、反射神経に自信があるなど、プレイヤーごとの得意を活かせる複数の攻略余地を残す。
- [必須] 全員が全ステージを簡単にクリアできる必要はない。誰かには苦手でも、別のプレイヤーの得意が刺さるステージを用意し、プレイヤー同士で攻略や見せ場の違いが出るようにする。
- [必須] クリアできない理由が分からない理不尽なステージではなく、苦手な動きや判断が見え、得意な人なら鮮やかに突破できるステージを目指す。

## ゲーム開始からクリアまでの構成方針

- [必須] プレイヤーは、各ステージ内で危険に踏み込むタイミングを選び、ゲーム全体では自分の得意が活きるルートを選ぶ。
- [必須] 苦手ステージを避けられる構成を検討する。ただし単なるスキップではなく、得意なルートを選んで進んでいる感覚を優先する。
- [必須] ステージ群は、目押し型、狭い通路の操作型、待ちと観察型、一気抜け型、休憩地点探し型、欲張り判断型など、性格の違うステージを組み合わせる。
- [未決定] 基本構成は、ステージをクリアすると次に進めるステージが2つか3つ開くワールドマップ式の分岐、または自由ステージ選択寄りの構成を有力候補にする。
- [未決定] ステージ内の複数ゴール、隠し分岐、スキップ権は面白い候補だが、主構造にするか補助にするかは慎重に決める。

## 実装方針

- 内部解像度は `1920x1080`。
- 実ウィンドウにはレターボックス付きで拡大縮小して描画する。
- 外部画像やフォントを使わず、1x1 テクスチャを拡大して矩形だけで描画する。
- 状態表示はウィンドウタイトルと簡易 HUD で行う。
- WASD、矢印キー、Xbox コントローラー左スティックに対応する。

- [未決定] 宝石をすべて取得したあとにゴールへ到達できることを、手作業またはプログラムで確認する。
- [未決定] ステージ作成時は、主導線だけでなく、プレイヤーサイズ込みで全宝石と出口へ到達できることを確認する。
- [未決定] 壁をジグザグに置く場合、横壁と縦壁が接続して通路を塞がないようにする。
- SpriteFont を追加して、画面内に操作説明とクリア表示を出す。
- 最初のフリー素材または仮素材を追加する。
- [対応済み] ステージデータを `StageDefinitions.cs` に分離した。
- ハイスコアやクリア時間の保存を検討する。
