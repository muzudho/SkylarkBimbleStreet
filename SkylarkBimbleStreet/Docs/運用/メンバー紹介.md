## Codex

Codex は開発補助として参加する。最終的な採用判断はむずでょさんが行う。

### 得意なこと

- プログラミング: MonoGame / C# の実装、設計、リファクタリング、バグ修正。
- ゲーム実装: 入力処理、画面遷移、セーブ処理、UI、当たり判定、簡単なゲームループ設計。
- ドキュメント作り: 要件整理、企画メモ、仕様書、作業ログ、README、素材ライセンス表、開発手順、リリース手順。
- ゲーム企画の壁打ち: 制約から現実的なゲーム案を出す、スコープを削る、即売会向けに遊びやすくする。
- ストーリーと台詞: 日本語の短いシナリオ、キャラ設定、イベント文、チュートリアル文、UI 文言、エンディング文。
- UI 作成: MonoGame 上のメニュー、タイトル画面、ポーズ画面、HUD、設定画面などの設計と実装。

### ある程度できること

- 画像素材作り: ラフ画像、背景案、キャラ案、アイコン案、テクスチャ案。
- キャラクターデザイン: 設定案、見た目の方向性、服装、色、性格、役割、画像生成用プロンプト作成。
- 音の当てはめ: 場面ごとに必要な効果音や BGM を考える、鳴らす場所を決める、MonoGame に組み込む。
- 素材探しの方針作り: 無料素材のライセンス確認観点、素材管理表、クレジット表の設計。

### 苦手なこと、直接は難しいこと

- 高品質な BGM 音声ファイルを直接作ること。曲調やループ BGM の要件作成はできる。
- 効果音ファイルをゼロから直接作ること。必要な効果音リストや無料素材の組み込みはできる。
- 完成品レベルの大量アート制作。1枚絵や案出しはできるが、統一感のある大量スプライトやアニメーションはスコープ管理が必要。

## このプロジェクトでの現実的な担当

- Codex は、プログラム、仕様整理、企画案、台詞、UI、素材管理、実装計画を主に担当する。
- 画像はラフ、仮素材、方向性検討から始める。
- 音は、必要な素材を決めて、ライセンスの明確な無料素材を組み込む方向を基本にする。
- 有料素材や有料ツールが必要な場合は、むずでょさんに相談してから進める。
