# ゲーム案メモ

記録日: 2026-06-10

## 前提

これは確定要件ではなく、初期要件を元にしたゲーム案のメモ。

重視する条件:

- 1プレイ 1分から3分で遊べる。
- 即売会で試遊しやすい。
- 無料素材または簡易自作素材で成立しやすい。
- MonoGame 未経験でも段階的に作れる。
- 230時間から460時間の中に学習時間も含めて完成を狙える。
- Windows 11 / デスクトップ PC / ノート PC 試遊に向く。
- 技術を剥がしても残る遊びの核がある。プレイヤーの判断、練習、上達を説明できる。

## ステージ目標の考え方

各ステージには、単一の「クリアー」だけではなく、複数の小目標を置く。

ただし、現状の試作のように「宝石全部回収でゴール開放」にすると、ゴール到達と宝石全部回収が同じ条件になってしまう。
この場合、［通過］と［宝石全部回収］を別目標として表示すると、プレイヤーが「宝石全部回収に加えて通過も必要なのか」と誤解しやすい。

［通過］を独立した目標にするなら、出口は最初から使える、または最低限の鍵だけで開くようにする。
宝石は出口開放の必須条件ではなく、追加評価や収集目標として扱う。

［通過］の具体案として、ステージ内にバス停を置く。
プレイヤーがバス停で一定時間待つとバスが来て、乗車すると宝石が未回収でもそのステージを［通過済み］にできる。
これは負けやスキップではなく、正式な通過手段として扱う。救急車がミス時に運ばれる演出なら、バスはプレイヤーが自分の意思で待って乗る対の演出になる。

病院とバス停は小さな記号ではなく、大きなランドマークとして描く。
病院は赤十字マークを使わず、窓のある白いビルと救急車が入る入口で表現する。
ミス時は救急車でスタート地点へ戻すのではなく、病院へ運んで降ろす。病院の近くにバス停を置くことで、プレイヤーは再挑戦するか、バスで［通過］するかを選べる。
その分ステージに使える面積は少し減るが、今の画面は十分広いため、施設が大きいほうが役割を理解しやすい。

例:

- 通過: ゴール地点まで到達する。宝石を全部取らなくても達成できる。
- 宝石全部回収: ステージ内の宝石をすべて取ってから通過する。
- 無傷: 障害物や敵に当たらずに通過する。
- 急速: 指定時間以内に通過する。

基本表現は「ステージをクリアーする」よりも「ステージを通過する」に寄せる。
宝石を全部取れなかったり、無傷や急速を達成できなかったりしても、ゴールまで行けたなら「負けた感じ」になりにくい。

表示上は、［通過］を他の目標の必要条件に見せない。
［通過済み］はステージを先へ進めるための状態、［宝石全部回収］［無傷］［急速］は追加達成の記録として分ける。

この設計により、初心者はまず通過を目指し、慣れたプレイヤーは宝石全部回収、無傷、急速などを重ねて狙える。
同じステージでも、通過重視、回収重視、安全重視、速度重視という遊び分けが生まれる。

## 有力候補

### 1画面アクションパズル

1920x1080 の1画面内で完結するゲーム。プレイヤーを動かして、敵、弾、床、スイッチ、ゴールなどを扱う。

例:

- 60秒以内に部屋から脱出する。
- 毎回配置が少し変わる。
- 1ステージ1分で終わる。

向いている理由:

- 即売会試遊に強い。
- 無料素材でも作りやすい。
- MonoGame の基本である描画、入力、当たり判定、音、ステージ管理を学べる。
- ステージ追加で拡張しやすい。

### 短時間スコアアタック

1分から3分で終わる、避ける、集める、壊す系のゲーム。

例:

- 夜の商店街で光るアイテムを集める。
- 制限時間内にできるだけ配達する。
- 障害物を避けながらスコアを稼ぐ。

向いている理由:

- ゲーム内容が未定でもプロトタイプを作りやすい。
- セーブはハイスコアだけで済む。
- Xbox コントローラー、キーボード両対応にしやすい。
- 即売会でプレイヤー交代しやすい。

## その他の候補

### 小さなローグライト風ミニゲーム

毎回ランダムに部屋やイベントが出て、3分で1周するゲーム。

例:

- 3分だけ探索する。
- 3つの選択肢を選びながら進む。
- 最後にスコアや称号が出る。

注意:

- 素材量を抑えつつリプレイ性を出せる。
- ランダム生成やバランス調整で難度が少し上がる。

### 日本語テキスト中心の選択式ゲーム

短い選択と結果で遊ぶゲーム。ノベルゲームほど大きくしない。

例:

- 即売会で3分だけ店番する。
- ひばりが丘近くの不思議な通りを歩く。
- 選択肢でスコアやエンディングが変わる。

注意:

- 日本語のみの方針と相性がよい。
- 素材が少なくても成立する。
- ゲームとしての操作感は弱くなりやすい。

### 盤面系ミニゲーム

グリッド上でキャラやブロックを動かすゲーム。

例:

- 荷物を押してゴールに置く。
- 敵の視界を避けて進む。
- 毎ターン1手ずつ動く。

注意:

- 実装が整理しやすい。
- 無料素材や自作ドットでも成立する。
- 派手さは後から足す形になる。

## いま一番現実的な方向

現時点では、`1画面アクションパズル` または `短時間スコアアタック` が最有力。

理由:

- まず完成形まで持っていきやすい。
- MonoGame の基礎を学ぶ題材としてちょうどよい。
- 試遊時間の条件に合う。
- 素材コストを抑えやすい。

## 仮の外枠

- ジャンル: 1画面アクションパズル、または短時間スコアアタック。
- 1プレイ: 60秒から180秒。
- 画面: 1920x1080 基準。
- 入力: Xbox コントローラー優先、キーボード対応。
- 言語: 日本語。
- 素材: 無料素材または簡易自作素材。
- セーブ: ハイスコア、設定、試遊用リセット。
- 完成目標: 2026年11月頭。
